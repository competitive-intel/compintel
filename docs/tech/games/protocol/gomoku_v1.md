# 五子棋交互协议 v1

## 编码与生命周期

协议使用 UTF-8 文本和十进制整数，字段之间由一个或多个 ASCII 空白字符分隔。用户程序在一局中只启动一次，必须持续从标准输入读取，并在每次轮到自己时向标准输出写出一行走法并刷新缓冲区。调试信息只能写到标准错误。

评测器在游戏结束后直接终止程序，不发送结束标记。评测器也不会把越界、已被占用或游戏结束后的对手走法发送给用户程序。

## 初始化

程序启动后首先收到三行：

```text
1
<height> <width>
<seat>
```

- `1` 是协议版本。
- `height` 和 `width` 分别是棋盘的 X、Y 方向大小；当前规则固定为 `15 15`。
- 坐标从 `0` 开始，合法范围分别是 `0 <= X < height`、`0 <= Y < width`。
- `seat = 0` 表示执白并先手；`seat = 1` 表示执黑并后手。这是本游戏版本的固定席位定义。

先手程序在读完初始化信息后立即输出第一步，此前不会收到一个虚构的对手坐标。后手程序会在初始化信息之后继续收到先手的合法走法，然后再输出自己的第一步。

## 每回合

除先手的第一回合外，每次轮到程序行动前，评测器输入对手的上一步：

```text
<X> <Y>
```

程序必须输出且只输出两个整数，并以换行结束：

```text
<X> <Y>
```

输出坐标必须在棋盘内且对应空位。额外 token、非整数、越界坐标和重复落子均判为 `INVALID_MOVE`。程序必须显式刷新标准输出，例如 C++ 可使用 `std::endl` 或在输出 `\n` 后调用 `std::flush`。

## 后手示例

执黑后手的程序收到初始化和白方的第一步：

```text
1
15 15
1
5 5
```

程序可以响应：

```text
5 6
```

若游戏尚未结束，下一次输入将只有白方的新坐标，不会重复初始化信息。

## 规则与限制

- 双方交替落子，最先在横、竖或任一斜线形成至少五个连续棋子的一方获胜；棋盘填满仍无人获胜则为平局。
- 不使用禁手规则。
- 每一步 CPU 时间上限为 `100ms`。
- 单个程序一局累计 CPU 时间上限为 `5s`。
- 每一步另设 `1s` 墙上时间保护，墙上时间不替代 CPU 时间判定。
- 单步标准输出最多 `64` 字节，并且必须在限制内输出换行。
- 当前沙箱同时限制为 `256MiB` 内存、`128MiB` 栈和最多 `8` 个进程。

评测通过 go-judge 的 turn-control 执行：程序仅在自己的行动回合中解冻运行，输出完整一行后立即再次冻结。因此等待对手期间不消耗程序的 CPU 或活动墙上时间预算。
