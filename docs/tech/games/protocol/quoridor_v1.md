# 路墙棋交互协议 v1

本文档定义 `game.slug = quoridor` 的规则坐标、标准输入输出格式和非法操作判定。协议中的所有整数使用十进制，字段以空白分隔，每一步占一行；程序输出后必须立即刷新标准输出。

## 棋盘与坐标

- 棋盘包含 9×9 个棋格，棋格坐标为 `(X,Y)`，左上角是 `(0,0)`，`X` 向右增加，`Y` 向下增加。
- 席位 0 是先手，棋子初始位于 `(4,0)`，目标是抵达任一 `Y=8` 的棋格。
- 席位 1 是后手，棋子初始位于 `(4,8)`，目标是抵达任一 `Y=0` 的棋格。
- 每方初始拥有十面墙。墙长为两个棋格，一旦放置便不能移动或收回。

墙使用四个棋格之间的内部交叉点作为中心坐标，因此 `X,Y` 的合法范围均为 1 到 8：

- 竖墙 `(X,Y,0)` 阻断 `(X-1,Y-1)` 与 `(X,Y-1)` 之间、以及 `(X-1,Y)` 与 `(X,Y)` 之间的移动。
- 横墙 `(X,Y,1)` 阻断 `(X-1,Y-1)` 与 `(X-1,Y)` 之间、以及 `(X,Y-1)` 与 `(X,Y)` 之间的移动。

## 初始化

程序启动后读取一行：

```text
1 S
```

`1` 是协议版本，`S` 是己方席位，只能为 `0` 或 `1`。席位 0 收到初始化后立即输出第一步；席位 1 继续读取席位 0 的第一步，然后才输出。

## 逐回合通信

除先手的第一步外，每次轮到己方时：

1. 从标准输入读取对手刚完成的一步；
2. 更新本地局面；
3. 向标准输出写入己方一步和换行，并立即刷新。

一步有两种格式：

```text
0 X Y
1 X Y O
```

- `0 X Y`：把棋子移动到棋格 `(X,Y)`。
- `1 X Y O`：以内部交叉点 `(X,Y)` 为中心放墙；`O=0` 为竖墙，`O=1` 为横墙。

评测端只接受对应格式所需的全部字段，不接受额外调试文本。标准错误可用于调试，但仍受沙箱输出限制。

## 合法棋子移动

棋子不能穿墙或落在对手棋子所在格。合法目的地按以下规则计算：

1. 若某个上下左右相邻格没有墙阻挡且不是对手位置，可以移动到该格。
2. 若对手位于没有墙阻挡的相邻格，并且沿同一方向的后方格仍在棋盘内、且对手与后方格之间没有墙，则可以直线跳到后方格。
3. 若上述后方格越界或被墙阻挡，可以从对手位置斜跳到其两侧棋格；对手与该侧棋格之间也不能有墙。

## 合法墙放置

放墙方必须仍有墙可用。新墙还必须同时满足：

- 中心坐标和方向合法；
- 不与同方向的已有墙发生任何一格长度的重叠；
- 不在同一中心点与另一方向的墙交叉；
- 放置后，双方棋子都至少仍有一条仅考虑墙阻挡、可以到达各自目标边的路径。

相邻但不重叠的墙合法。路径检查不把棋子视为永久障碍，因为棋子会继续移动。

## 终局与异常

任一棋子首次抵达自己的目标边时立即获胜，对局结束，双方输入流随后关闭。路墙棋没有和棋终局。

格式错误、越界、非当前行动方、非法移动、墙数量耗尽、墙重叠或交叉、以及封死任一方全部路径，都会使该步非法。用户程序的非法操作记为 `INVALID_MOVE`；平台内置程序的非法操作属于平台配置错误，记为 `INTERNAL_ERROR`。

## 资源语义

双方是两个独立的长驻 C++20 进程。每一步只解冻当前行动方，并分别限制单步 CPU、单方整局累计 CPU、单步墙上时间、内存、输出和进程数。具体 CPU 与内存限制来自游戏目录；单步墙上时间仅用于阻止永久阻塞，不能替代 CPU 限制。
